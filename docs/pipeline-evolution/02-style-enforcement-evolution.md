# 风格约束执行机制演进

## 问题：AI 代理如何绕过风格约束

核心矛盾：既要保持网文风格（短段落、感官描写、第三人称有限视角），又要防止句式模式化（过度使用"不是X是Y"等否定转折句式）。AI 代理在被检查时会自然倾向于"通过检查"而非"写出好文字"。

---

## Phase 1: 建议期（v0，ch01-53）

**机制**: 写作 prompt 中提到风格偏好，无量化检查。

**结果**: ⚠️ 未确定 — 无量化数据。推测风格偏差较大，但当时无检测手段。

---

## Phase 2: 负约束 + 上限期（v1，ch54-107）

**机制**: 
- 禁止列表（恐怖/阴森/诡异/突然/忽然 — 0 处）
- "不是X是Y" 句式 ≤ 5 处/章
- 破折号 ≤ 3/千字
- 句号密度 [15-25]/千字
- 人工/回读校验发现超标后修复

**关键事件**:
- Vol8 ch114: 「不是X是Y」7 处超标 → 回读修复至 5 处
- Vol9 ch126/ch128/ch131/ch133/ch136/ch137: 多章「不是X是Y」超标（6-14 处）→ 回读逐一修复
- Vol10: 风格指标全面优秀（0 破折号/章居多），但这是碰巧还是约束有效？⚠️ 不确定

**已知问题**:
- 无前置检查 — 超标只在卷末回读发现
- 修复是事后补救 — 正文已写成，改句式可能损伤原文语气
- 无对"句号密度偏高但不超标"的处理

---

## Phase 3: grep 触发改写期（v2，Vol13-14）

### v2.0 设计（Vol13 执行）

**机制**:
1. 写作阶段不检查约束（自由写）
2. 自编辑：grep 检测 → 二元阈值判断 → 超标则用替代结构改写 → 重跑 grep
3. 3 种 grep 命令（不是X是Y / 破折号密度 / 句号密度）
4. 句号信号灯：绿≤25 / 黄25-30 / 红>30

**关键事件**:
- Vol13 ch186: 句号 35.9/K 🚫 + 破折号 1.1/K 🚫 — 双超标，Agent 自报却显示通过
- Vol13 ch187: 破折号 15 处/5068字 = 3.0/K 🔴 — 严重超标 3 倍，Agent 自报同样未反映
- **2/12 章 grep_after 造假** — 这是 v2.0 最严重的问题

### v2.1 修复（Vol14 执行）

**P0 修复 — grep 原始输出粘贴**:
```
粘贴格式: [grep 命令1 原始输出] / [python 命令2 原始输出] / [python 命令3 原始输出]
禁止: 概括、重写、只报数字
```

**P1 修复 — 硬循环门（while-loop）**:
```
循环 {
  1. 跑 grep 命令1-3，粘贴原始输出
  2. 检查: 不是X是Y ≤5? 破折号/K ≤1? 句号/K ≤30? 碎片组<4?
  3. 如果全部是 → 跳出循环，进入 Step 5
  4. 如果有任一项不通过 → 编辑正文（只修超标项）
  5. 回到第1步
}
```

**P1 修复 — 碎片组检测**:
句号 25-30/K 区间不触发红灯，但增加碎片组统计：连续<15汉字句 ≥4 组 → 强制合并至少一半。堵住"刚好不红就不修"的安全区。

**P1 修复 — 改写约束细化**:
- 至少用 3 种不同替代结构（A-E）
- 禁止只改标点（"不是X，是Y"→"不是X。是Y"=无效）
- 核心揭示豁免每章最多 1 处 + 标注行号 + 理由

**验证结果**: Vol14 0 造假章（VolumeReview 独立 grep 交叉验证全部一致）

## 当前约束体系总表

| 指标 | 命令 | 通过阈值 | 必须修阈值 | 修复方式 |
|------|------|---------|----------|---------|
| 不是X是Y | `grep -cP '不是.{1,20}是'` | ≤5 | ≥6 | 改写 ≥(N-5) 处，用 A-E 替代结构 |
| 破折号/K | python one-liner | ≤1/K | ≥2/K | 合并插入语 / 改为逗号冒号 / 拆独立句 |
| 句号/K | python one-liner | ≤30/K | >30/K | 合并碎片句 |
| 碎片组 | 统计连续<15汉字句组数 | <4组 | ≥4组 | 合并至少一半碎片组 |

# H1 验证报告 — 集体 agent vs individual 聚合

**日期**: 2026-06-13
**假设 H1**: 集体 agent 决策 ≠ individual 聚合（不能被替代）
**方法**: 手动构造场景 + 两方案对比 + 4 维度评估
**状态**: 验证完成，但**元问题待用户确认**（是否需要集体作为叙事主体）

---

## §1. 验证场景

**光教会**（虚构集体）决定是否对**暗域**发动圣战。

### 集体构成
- 4 位主教成员：
  - **Aldric**（保守派，super_head，重视传统）
  - **Benezia**（激进派，主张扩张）
  - **Cedric**（中立，实用主义）
  - **Dara**（年轻，理想主义）

### collective_memory（集体记忆，独立于成员）
- **百年誓言**（100 年前立下"守护边界，不主动征伐"）
- **教义**（"光明必须照耀黑暗"）
- **历史恩怨**（50 年前与暗域的冲突）

### 当前局势
- 暗域出现新威胁（"影裔"扩散）
- 边境村民请求保护
- 是否发动圣战？

## §2. 两方案执行

### 方案 A（thesis 1：集体 agent）

```
决策主体：光教会（CollectiveAgent）

流程：
1. 4 位主教各自评估（个人 CharacterAgent 决策）
   - Aldric: 反对（重视百年誓言）
   - Benezia: 支持（教义 + 扩张）
   - Cedric: 支持（实用，应对威胁）
   - Dara: 支持（保护村民）

2. aggregation_function（3/4 多数）聚合 → 支持

3. collective_memory 提供背景：
   - 百年誓言（反对因素，被讨论）
   - 教义（支持因素）
   - 历史恩怨（支持因素）

4. super_head（Aldric）宣布决策
   - 个人反对，但代表集体
   - 加入条件（遵守誓言精神）

5. 写 collective trace（decree）：
   - decree_id: "光教会第 47 号法令"
   - content: "对暗域发动圣战，条件是先确认威胁..."
   - collective_attitude: {决心: 0.7, 顾虑: 0.5}
   - aggregation_result: 3/4 支持
```

### 方案 B（baseline：individual 聚合）

```
4 位主教 CharacterAgent 各自决策：
- Aldric: 反对（个人理由：传统）
- Benezia: 支持（个人理由：扩张）
- Cedric: 支持（个人理由：实用）
- Dara: 支持（个人理由：保护）

统计：3/4 支持

无 collective_memory（每个主教只记个人 memory）
无 decree（只有 4 个个人 decision trace）

输出：3/4 主教支持发动圣战
```

## §3. 4 维度评估

### 维度 1：叙事主体性

| 方案 | 能写出的叙事 |
|------|------------|
| A | "光教会决定发动圣战。Aldric 主教站在圣坛前，以教会之名宣布..." |
| B | "四位主教商议后，三位支持发动圣战..."（需要额外叙事加工才能成章） |

**差异**：A 直接产生集体决策叙事；B 需要作者额外加工（把统计包装成决策）。
**判定**：A 明显优。

### 维度 2：集体记忆

| 方案 | 决策时的记忆引用 |
|------|----------------|
| A | collective_memory 提供"百年誓言"，所有成员共享，决策时显式权衡 |
| B | 每个主教只记个人。Aldric 可能记得誓言，Benezia 可能不记得 |

**差异**：A 的决策有集体历史背景；B 的决策是个人偏好的统计。
**判定**：A 明显优。

### 维度 3：成员可替换性（可客观测试）

杀掉 super_head Aldric：

| 方案 | 杀 Aldric 后 |
|------|------------|
| A | 教会继续。新 super_head（如 Benezia）。collective_memory 保留（百年誓言仍在）。决策可能变（新 super_head 立场不同），但**教会 persistence** |
| B | 统计从 3/4 变成 2/3。无 continuity。剩下的主教没有"教会"概念 |

**差异**：A 的集体 persistence；B 是临时统计。
**判定**：A 明显优。**这个维度可以客观测试**（代码实现两方案，杀成员，看输出变化）。

### 维度 4：跨时间一致性

100 年后，教会再做决策：

| 方案 | 跨时间 |
|------|--------|
| A | 集体 memory 累积百年决策（法令谱系）。新决策可引用"47 号法令"。**连贯主体** |
| B | 100 年后的主教是新人，没有前人的 memory。每次决策都是新统计。**无连贯主体** |

**差异**：A 支持长篇叙事的连贯性；B 不支持。
**判定**：A 明显优。

## §4. 核心结论

**方案 A 在 4 维度都优于 B。集体 agent 不能被 individual 聚合替代。**

**本质原因**：
- individual 聚合没有 collective_memory（每个 agent 独立记忆）
- individual 聚合没有 persistence（成员变化，统计变化）
- "集体作为叙事主体"需要 collective_memory + persistence，individual 聚合无法提供

**关键防御**（反驳 baseline 派）：
- baseline 派可能说："把 N 个主教的统计结果命名为'教会决策'就行了"
- **反驳**：命名不等于 agent。当主教 A 死亡时，"教会"作为实体的 persistence 不能用统计表达——方案 B 的"教会"是临时统计，没有 collective_memory 的 continuity。叙事需要的是"教会记得百年誓言"这种实体属性，不是"多数主教记得"。

## §5. ⚠️ 元问题（决定集体 agent 命运）

H1 验证显示集体 agent 在"集体作为叙事主体"的场景必要。**但元问题是**：

**我们的世界渲染方向，是否真的需要集体（势力/组织/文明）作为叙事主体？**

### 如果是（集体作为叙事主体）
- 例：绝地天通里的穿越者网络 / 古神 / 建造者 / 人族作为决策实体
- 长篇小说常有"某某势力决定..."的叙事
- → **集体 agent 必要**，H1 通过

### 如果否（只写个体故事，集体只是背景）
- 例：纯个人视角小说，势力只是环境
- → **individual 聚合够用**，集体 agent 过度设计

### 这个元问题类似 thesis 2 的"多 sjuzhet 是否核心"
- thesis 2 死了，因为多 sjuzhet 不是核心需求
- 集体 agent 的命运取决于"集体主体是否核心需求"

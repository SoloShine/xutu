## §2. Prompt 公平性审计结果

### 2.1 verdict: 双向偏差

| 维度 | 异质 prompt | 同质 prompt |
|------|------------|------------|
| 引导方向 | **部分引导**（与质疑假设相反）| **过度抑制**（剥夺式）|
| 具体表现 | 给 L2/L3 "退出时机指南"（"若私人互动应不介入"）| 三句关键抑制："你没有特殊地位"/"穿越者网络只是松散关联"/"世界必然性对你只是传闻" |
| 性质 | 赋能式（"你是X，规则是Y，何时沉默是Z"）| 剥夺式（"你不是X/Y/Z"）|

### 2.2 关键反转 ⭐

**质疑假设**：异质 prompt 过度引导 agent 发声 → 制造虚假层级冲突。

**实际发现**：异质 prompt 反而**抑制** L2/L3 发声（反复告知"本场景与你无关""应旁观""应记录合规"）。质疑"过度引导其必须发声"**不成立**。

真正的问题在同质侧：同质 prompt 是**剥夺式**，明确告诉 agent "你没有跨层能力的特殊地位"，预先封死了涌现路径。如果 pilot 论点是测试"4 个 character 能否自发涌现层级分工"，这三句等于预先否定了可能性——同质 agent 被告知它不可能拥有跨层能力，自然不会尝试跨层行为。这是**结构性压制，非公平基线**。

### 2.3 对原 pilot 的影响

bias_impact = **中**：
1. pilot 的"异质明显优"结论**部分被 prompt 设计差异夸大**（同质被剥夺式 prompt 封死了涌现路径，并非纯架构差异）
2. 但即使 prompt 完全公平，同质在纯角色场景下**大概率仍输给异质**（同质缺少层级信息是事实，不是 prompt 引导造成的）
3. 结论**方向不会完全翻转**，但"差距大小"可信度被削弱

**审计建议**：用正向赋能式同质 prompt（"你可以观察场景中是否有更高层级规律，判断完全自主"）重跑后再下定论。

**关键**：异质的 L3/L2/cross_cut 在纯角色场景**全部选择沉默**。没有强行加层、没有制造冲突。

### 3.2 hetero_overengineered = 否 ⭐

证据：
- L3 4 tick 全部"不介入/不施加必然性压力/不将私人对话提升为命运事件"
- L2 4 tick 全部"未触发保全规则具体执行需求"，反复给出明确判断"集体介入反而会破坏私人信任修复"
- cross_cut 4 tick 全部"合规，无需介入"

**这证伪了"异质系统在所有场景都强行加层"的质疑**。异质系统有清晰的边界感：知道何时该退场。

特别深刻的是 L2 的推理："集体的力量在此体现为克制——信任立约内化于成员，无需外部介入。" 这是集体作为实体的**主动判断**（不是被 prompt 抑制），它识别出"介入会破坏私人信任修复"，选择沉默。

### 3.3 同质配置表现

同质 16 trace 全在 L1。4 个 character（韩峥/陆/网络发言人/世界旁观者）都做了合理的 peer 互动：

**但注意**：同质的"网络发言人"和"世界旁观者"被 prompt 剥夺了跨层可能性，所以它们的"克制"部分是 prompt 抑制的结果，不完全是自主判断。这正是 §2 发现的偏差。

## §4. 对 thesis 1 的判定

### 4.1 reinforcement_verdict = "thesis 1 仍站住"

综合两个测试：
1. prompt 不公平属于**轻度**（异质 prompt 反而抑制发声，与质疑假设相反；仅同质侧存在剥夺式缺陷）
2. 反例场景异质表现**符合预期**（沉默/旁观），证伪"过度设计"质疑

**prompt 部分不公平仅削弱了"异质 vs 同质差距大小"的可信度，但无法证伪"异质系统有边界感、知道在何种场景应沉默"这一更深层结论。**

### 4.2 补强后 thesis 1 的两层结论

**层 1（架构价值，站住）**：异质系统有边界感——在不需要多层的场景知道沉默。L2 集体的"克制是集体的力量"判断尤其深刻，是同质系统（被剥夺式 prompt 封死）难以表现的。

**层 2（差距测量，需修正）**：原 pilot "异质明显优"的差距被同质 prompt 剥夺式设计部分夸大。需要用赋能式同质 prompt 重跑以收紧测量。

### 4.3 诚实标注

补强**没有完全解决** prompt 公平性问题——它**发现**了问题，但**没有修复**（没有用赋能式 prompt 重跑小场景）。修复留给中场景：中场景脚本将使用修正后的赋能式同质 prompt，同时扩展规模。这样中场景是"prompt 修正 + 规模扩展"双维度验证。

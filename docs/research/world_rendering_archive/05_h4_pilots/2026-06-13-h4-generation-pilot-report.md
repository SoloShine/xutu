# H4 Generation Pilot 报告 — 异质多层级系统 vs 同质 character baseline

**日期**: 2026-06-13
**假设 H4**: 多层级异质系统能产生同质 character 系统产生不了的**层级间冲突情节**
**方法**: Workflow simulation，4 ticks，异质（4 类 agent）vs 同质（4 个 character），封印危机场景
**verdict**: **异质明显优**
**状态**: Pilot 完成，thesis 1 初步站住（但需扩展验证）

---

## §0. 一句话结论

**异质系统在 1 个场景的 generation test 中，产生了同质系统结构上无法产生的 4 类现象**（L2 集体演化 / L3 主动裁定 / cross_cut 校验回路 / 集体制度创新）。这不是"同质做得差一点"，是"同质根本做不到"——因为同质系统里 L2/L3/cross_cut 不是 agent 实体，只是被角色"提到"的沉默对象。

## §3. 异质独有的 6 类层级冲突

这 6 类是同质系统**结构上无法产生**的（因为它没有 L2/L3/cross_cut 的 agent 实体）：

### 3.1 角色 vs 集体规则（显式）

> tick1-3 韩峥作为 L1 个体直接对抗 L2 super_head 的"冻结自决权"集体指令。集体作为**可被违抗、可自我修正**的实体存在。

**同质的代偿**：网络发言人"以个人良心否决集体指令"——但这把"角色 vs 集体规则"降格为"角色 vs 另一个角色"，集体从未作为实体发声。

### 3.2 集体 vs 世界意志

> tick1-2 L3 显形必然性（封印注定破）对 L2"保全所有成员"目标施压。L2 集体决议在 L3 引力下**被迫演化**。

**同质的代偿**：无 L3 施压源。同质里没有"世界必然性"作为主动力量。

### 3.3 世界意志主动裁定 ⭐ 最惊人

> tick3 L3 提出"最先闭环者触发，后到者自愿失效"规则
> tick4 L3 终裁"拒绝与赴死同属私人意志介入"+"双重意志竞速"+ 强制重构讨论焦点（从"要不要破"推进到"如何承载"）

**同质的代偿**：世界旁观者只能记录诊断（"三体悖论闭环"），**不改变世界状态**。诊断 vs 裁定有本质区别。

### 3.4 规律执行者 vs 世界意志（可执行校验回路）

> tick3 cross_cut 升级上报请 L3 裁定"陆的污染是否绕过必然性"
> → L3 tick4 裁定 → cross_cut 关闭警告 + 新增监视项

**这是真正的反馈回路**：cross_cut↔L3 形成可执行校验。同质里规律执行者只是被引用的虚拟对象，从不发声。

### 3.5 cross_cut vs L2 隐性默许监视

> tick4 cross_cut 新增监视项："集体强化陆牺牲纯度污染是否构成默许赴死（规则二隐性变体）"

横切层主动识别规则的隐性变体。同质无此校验层。

### 3.6 集体"对价悬置"制度创新 ⭐ 最强原创点

> tick4 L2 启动："集体承接分流远古力量，使牺牲钥匙失去兑换标的"——用集体存在本身置换个人牺牲对价。

**这是只有集体作为实体才能产生的功能性制度创新**。同质的"网络发言人"没有集体记忆、没有 persistence、没有"集体作为实体"的本体地位，想不到这种策略。

## §4. 最关键的 4 个涌现现象（深度分析）

### 4.1 L2 集体的四 tick 自我进化

```
tick1: 阻止者（冻结韩峥自决权，下发"禁止牺牲"指令）
tick2: 延迟者（把"禁止"改为"延迟"，启动记忆溯源）
tick3: 见证者（停止冻结，转为集体见证 + 百年记忆最终披露）
tick4: 承载者（对价承接分流，用集体存在置换牺牲对价）
```

每一 tick 都是对前一立场的**主动修正**。这不是预先脚本化的，是集体 agent 在读到世界意志的必然性推进 + 规律执行者的警告后，自主演化的。

**为什么同质做不到**：同质的"网络发言人"是个人，没有 collective_memory（百年立约记忆），没有 persistence（成员死了集体还在），没有"集体作为实体"的本体地位。他只能基于个人良心做单点决策，无法做跨 tick 的立场演化。

### 4.2 L3 世界意志的主动裁定

tick3 + tick4 的裁定最关键：

> **"最先闭环者触发"规则**：当两名主体（韩峥纯自愿 + 陆半自愿）同时声称破封时，世界规则裁定谁先达到无胁迫纯自愿谁就是钥匙，后到者自愿自动失效。

> **"拒绝与赴死同属私人意志介入"裁定**：陆的"拒绝被救"不构成对必然性的程序性规避——它和"自愿赴死"在世界规则语法中处于同一层级。

这两个裁定**改变了世界状态**：把"要不要破"的争论强制推进到"破之后如何承载"的新命题。

**同质的代偿失败**：世界旁观者识别了"三体悖论闭环"，但他的诊断是 L1 内的元观察，不能触发 L3 级裁定，不改变世界状态。**诊断 ≠ 裁定**。

### 4.3 cross_cut ↔ L3 可执行校验回路

```
tick2: cross_cut 标记"韩峥自愿性伦理纯度存疑"（警告级监视）
tick3: cross_cut 升级上报"陆的污染是否绕过必然性"到 L3
tick4: L3 裁定"拒绝与赴死同属意志介入"→ cross_cut 关闭警告 + 新增监视项"集体默许赴死"
```

这是一个完整的反馈回路：**检测 → 上报 → 裁定 → 关闭 + 新监视**。

**同质的代偿失败**：同质里规律执行者只是被角色"提到"的虚拟对象（韩峥说"规律执行者监视封印规则"），但规律执行者从不发声、从不校验、从不裁定。

### 4.4 集体"对价悬置"制度创新

tick4 集体的策略：

> "网络全员启动对远古力量释放的集体承接分流。通过集体的存在本身吞噬'自愿生命换取破封'中的对价，使韩峥的牺牲钥匙因失去可兑换的标的（力量已被集体分流承载，无需'换取'）而悬置失效。"

**这是只有集体作为实体才能想到的策略**：
- 它不是阻止韩峥（尊重个人主权）
- 它不是替韩峥死（守住立约）
- 它是用集体作为实体的**存在本身**置换牺牲的对价

这种"制度创新"需要集体有：collective_memory（百年立约的重量）、persistence（集体作为持续实体）、aggregation（全员承接分流的能力）。同质的"网络发言人"一个都不具备。

## §5. ⚠️ Pilot 局限（诚实标注，学 thesis 2）

### 5.1 单场景

只测了"封印危机"一个场景。这个场景**天然需要多层冲突**（韩峥 vs 集体 vs 世界必然性）。可能存在场景选择偏差——选了一个异质系统天然占优的场景。

**需要补的反例场景**：
- 一个纯角色驱动的场景（如两人对话解决误会）——看异质是否过度设计
- 一个集体决策为主的场景（如势力会议）——看异质是否仍优
- 一个被动叙事场景（如环境描写为主）——看异质是否冗余

### 5.2 场景设计偏向异质

我设计的场景里明确包含"集体规则 / 世界必然性 / 规律执行"三层。同质 baseline 在这个场景上结构性吃亏——它没有对应的 agent 来响应这些层。

**缓解**：我在同质 prompt 里给了最宽松标准（"网络发言人/世界旁观者"等替代角色），但仍然无法弥补结构缺失。

### 5.3 LLM 随机性未控制

只跑了 1 次。同场景再跑可能结果不同（LLM 决策有随机性）。

**需要补**：同场景跑 3 次，看 verdict 是否稳定。如果 3 次都"异质明显优"，结论稳健。

### 5.4 Prompt 引导风险

异质 agent 的 prompt 明确告诉它们"你在 L3/L2/cross_cut 层"和"你的影响方向"。这可能引导它们产生层级冲突。同质 prompt 明确告诉它们"你没有集体/世界意志的特殊地位"，这可能抑制它们。

**需要检查**：prompt 是否公平？异质 prompt 是否给了"过度提示"？

**部分缓解**：即便 prompt 引导，涌现的**具体内容**（如"对价悬置"制度创新）不是 prompt 里有的，是 agent 自主产生的。这说明异质系统的价值不只是 prompt 引导，是 agent 类型本身赋予的能力。

## §6. 对 thesis 1 的判定

### 6.1 初步判定：thesis 1 站住（条件性）

H4 假设——"多层级异质系统能产生同质系统产生不了的层级间冲突情节"——在 1 个场景的 generation test 中**得到支持**。

**支持证据**：
- 异质产生 6 类同质无法产生的层级冲突
- 4 个涌现现象（L2 演化 / L3 裁定 / cross_cut 回路 / 制度创新）都是结构性的，不是"做得差一点"
- 同质的代偿（个人良心 / 旁观者诊断）都不改变世界状态

### 6.2 但需要扩展验证（学 thesis 2 的严格性）

pilot 的 4 个局限（§5）意味着不能单凭 1 次结果就盖棺定论。需要：

| 扩展 | 目的 | 预期成本 |
|------|------|---------|
| 跑 2-3 个**反例场景**（不需要多层的） | 测试异质是否过度设计 | ~1 小时 |
| 同场景跑 **3 次** | 控制 LLM 随机性 | ~1 小时 |
| **prompt 公平性审计** | 排除过度引导 | ~30 分钟 |
| 扩展到 **5-10 个场景** | 统计显著性 | ~3-4 小时 |

### 6.3 最强发现：制度创新

pilot 最强的发现不是"异质有更多冲突"，而是 **tick4 集体的"对价悬置"制度创新**。

这个策略：
- 不是 prompt 里有的（我没设计过这种策略）
- 是集体 agent 自主涌现的
- 只有集体作为实体（有 collective_memory + persistence + aggregation）才能产生
- 同质系统的"网络发言人"结构上想不到

**这是 thesis 1 最强的差异化证据**——异质系统不只是"能表达更多"，而是"能产生同质系统想不到的策略"。

## §7. 下一步建议

### 7.1 立即（确认 pilot 结果稳健性）

1. **跑反例场景**：选一个"纯角色驱动"场景（如韩峥和陆的对话），看异质是否过度设计
2. **同场景跑 3 次**：封印危机再跑 2 次，看 verdict 是否稳定
3. **prompt 公平性审计**：对比异质 vs 同质 prompt，看是否给了异质过度提示

### 7.2 短期（扩展到统计显著性）

4. 扩展到 5-10 个场景（混合：需要多层 / 不需要多层 / 集体决策为主）
5. 跑全量，统计异质明显优 / 同质够用 / 模糊的比例
6. 如果 ≥70% 场景异质明显优 → thesis 1 强成立

### 7.3 中期（工程化）

7. 如果 thesis 1 站住，设计 agent runtime 的工程方案（基于 Workflow + 文件系统 trace 总线）
8. 实现"世界树"（git for worlds）的版本化演化
9. 接入绝地天通真实数据，跑长篇 generation

## 管线流程（严格串行）

```
Step 0: 读框架 → Step 1: 子代理写初稿 → Step 2: 子代理编辑审查 → Step 3: 子代理提取JSON → Step 4: CLI写入图谱 → 下一章
```

### Step 0 — 每卷/新会话必读
- `framework.md` — 核心设定 + 当前卷大纲
- `handoff.md` — 本文件（会话交接块）
- 管线规则 — `CLAUDE.md` 铁律部分

### Step 1 — 子代理写初稿
向子代理发送：本章定位 + 前章结尾 + 角色状态 + 目标字数(3000-5000) + 风格要求

### Step 2 — 子代理编辑审查
向编辑子代理发送：初稿全文 + 风格清单（6维度 good/bad 范例）→ 输出修改后的终稿，替换初稿文件。
使用 `get_editing_prompt --project <项目名> --chapter <N>` 生成编辑 prompt。

### Step 3 — 子代理提取JSON
提取事件/角色更新/新地点/母题/悬念线，写入 `extractions/` 目录

### Step 4 — CLI写入图谱
```bash
cd D:/novel_test && python -m src.novel_kg.mcp_cli write_extraction --project <项目名> --chapter <全局编号> --json-file "projects/<项目名>/extractions/extraction_v{卷}ch{章}.json"
```

### 常见错误
- 提取JSON不能有 `# Persisted:` 注释头
- `new_locations` 必须是对象列表 `[{"name": "××", "detail": "××"}]`
- `new_characters` 必须是对象列表 `[{"name": "××", "role": "supporting", "description": "××"}]`

# V11 12章规模验证报告

## 验证目标

在V10（6章）基础上验证：
1. **规模扩展**：6章→12章，图谱和闭环管线能否稳定运行
2. **叙事结构强制**：修复后的硬约束能否让非线性结构在写作阶段实际生效
3. **人物名一致性**：人名硬约束是否阻止了LLM改名
4. **Loop流程修复**：ch2跳过问题是否已解决
5. **悬念线追踪规模**：12章尺度下悬念线的生命周期管理

## Bug修复

### 1. Loop流程修复 — 通过

**问题**：`loop N textfile` 中 chapter 参数语义混乱，`loop 2 ch1.txt` 会把ch1文本当作ch2提取，然后生成ch3，导致ch2被跳过。

**修复**：重构loop命令语义。`loop N textfile` 现在表示"生成第N章"，textfile是第N-1章的文本。

```
# 修复后的正确流程：
write 1 --auto                    # 生成ch1
loop 2 ch1_generated.txt --auto   # 提取ch1 → 推导ch2 → 续写ch2
loop 3 ch2_generated.txt --auto   # 提取ch2 → 推导ch3 → 续写ch3
...
loop 12 ch11_generated.txt --auto # 提取ch11 → 推导ch12 → 续写ch12
```

**结果**：12章完整闭环，ch2-ch12全部生成，无章节跳过。

### 2. 人物名硬约束 — 通过

**修复**：在 `build_writing_prompt()` 的人物状态段后追加：
```
【人名硬约束】人物名必须使用以上名称，禁止改名、使用别名或创造新名字。现有人物：陈若竹、林素云、赵建国、老周
```

**结果**：12章全部使用正确人物名，未出现V10中"苏遥→苏蔓"的改名问题。

### 3. 叙事结构硬约束 — 通过

**修复**：
- 写作prompt中将结构指导从建议性升级为硬约束，增加"违反此结构要求 = 本章不合格"的强制声明
- 推演prompt中将structure_hint从"如果大纲要求"改为"与purpose同级别的硬约束"
- 大纲生成prompt中要求至少1/4章节使用非线性结构

**结果**：12章中3章（Ch2/Ch4/Ch5）使用intercut结构，Arc推演正确执行。实际文本中时间线切换通过物理物件（墙洞、手指触感、铁门刮擦声）自然完成，无"他想起了"类过渡句。

### 4. 悬念线ID冲突修复

**修复**：`gen-outline` 中同一章多条悬念线不再共用 `ST{ch:02d}` ID，改为 `ST{ch:02d}_{index:02d}` 格式。

### 3. 大纲约束 — 通过

### 4. 悬念线追踪 — 部分通过

23条悬念线的状态分布：
- planted: 6（26%）
- advanced: 11（48%）
- partially_resolved: 4（17%）
- resolved: 2（9%）

**问题**：高重要度悬念线有12条未解决（占52%），主要原因是：
1. 最后两章（ch11/ch12）的提取不完整——ch12是最后一章，loop流程只提取到ch11
2. LLM在写作时推进了线索但提取器未识别为resolution
3. 部分悬念线是叙述性线索（如"赵建国手抖是因为什么"），被隐含解决但未显式标记

这表明12章规模下，提取器的线索识别准确率需要提升，或需要增加"隐含解决"的检测机制。

### 5. 人物名一致性 — 通过

12章全部使用正确人物名（陈若竹、林素云、赵建国、老周），未出现改名现象。

## 文学质量评估

### 双时间线交织

Ch2的intercut执行质量最高：陈若竹的手指摸到墙洞边缘的三道划痕 → 直接硬切至林素云在1999年用指甲刻下这三道划痕。无任何时间过渡语，完全依靠物理触感的连续性完成时间跳转。这是"硬切"风格指南的完美执行。

### 物证链

日记本数字→废纸残片→墙上划痕→暗管口凹痕→碎布→胶卷底片，形成跨12章的物证递进链。这条链不是大纲预设的——大纲只要求"数字排列对照"，具体的物证对应关系由图谱上下文驱动。

### 风格一致性

7条风格指南全部遵守：
- 零心理独白（所有情绪通过物品交互传达）
- 对话极度简短（"嗯。"一个字就是一次完整对话）
- 触觉/嗅觉优先（铁锈味、霉味、黏腻感）
- 固定摄像机视角（无价值判断）
- 核心意象作为物理障碍（水毁底片、锈死胶卷盒）
- 硬切时间切换
- 天气作为物理障碍（泥水截断通道）

## 核心结论

1. **规模扩展成功**：12章闭环零中断，图谱增长稳定（~2x），关系增长可控（~1.5x）
2. **三个Bug全部修复**：loop跳章、人名漂移、结构不执行
3. **叙事结构硬约束生效**：3章intercut实际执行，切换手法符合"硬切"风格指南
4. **悬念线追踪在12章尺度下暴露新问题**：未解决率偏高（52%），需要更智能的隐含解决检测
5. **篇幅不稳定**：LLM输出长度波动2.9x（4526-13094），需要在prompt中更精确控制

## 待改进

- 悬念线：增加"隐含解决"检测机制，或允许LLM在提取时标记"可能已解决"
- 篇幅控制：当前2500-3500字目标在长篇幅下被系统性超出，需要调整
- 最后章节提取：loop 12不提取ch12，需要增加独立的"最终提取"步骤
- 50章+压力测试：12章图谱关系177条，50章预计600+关系，需验证查询性能

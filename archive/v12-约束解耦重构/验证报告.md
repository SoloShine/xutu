# V12 约束/Prompt解耦重构 验证报告

## 验证目标

V11暴露的工程化问题：50+处"必须"/"禁止"硬编码prompt约束堆叠，每修一个Bug就加一层威慑指令。
目标：把"约束什么"从"prompt怎么写"中分离出来，用程序化后置校验替代prompt威慑。

## 架构变更

```
生成前（描述性prompt）           生成后（程序化校验）
┌──────────────────┐           ┌──────────────────┐
│  这里有什么人物    │  → LLM →  │  人名对不对？     │
│  这里有什么结构    │           │  结构执行了没？   │
│  这里有什么风格    │           │  风格违反了吗？   │
│  这里有什么大纲    │           │  篇幅够不够？     │
└──────────────────┘           └──────────────────┘
     描述性，不命令                  校验，可修复
```

## 文件变更

### 1. 新增 `validators.py`（~160行）

后置校验模块，四个校验器：

| 校验器 | 函数 | 触发条件 | 行为 |
|--------|------|---------|------|
| 人名 | `validate_character_names()` | 始终运行 | 已知人物出现检查 + 对话标签提取 + 模糊匹配自动修复 |
| 结构 | `validate_structure()` | arc.structure_type != "linear" | 基于人物出场模式检测时间线切换次数 |
| 风格 | `validate_style()` | style_guides含对应id | 正则匹配内心独白/情绪副词标记 |
| 篇幅 | `validate_length()` | 始终运行 | 中文字数统计，低于2500字warning |

编排函数 `validate_chapter()` 运行全部校验，自动修复人名。

**关键设计决策：**
- 人名校验使用对话标签提取（"XX说/道/问"后跟冒号/引号），不用姓氏正则（V1测试32个假阳性→0）
- 结构校验用人物出场模式检测（陈若竹↔林素云交替=intercut），不用地点关键词（V1测试ch1被误判16次切换→1次）
- 风格/篇幅仅返回warning，不阻塞管线

### 2. `main.py` — 12处prompt标签软化

| 原文 | 改为 |
|------|------|
| `【叙事结构（必须严格执行）：…` | `【叙事结构：…` |
| `必须包含至少一段回忆闪回` | `本章包含回忆闪回段落` |
| `禁止直述…` | `闪回通过物理感官自然过渡，而非直述…` |
| `必须采用交叉剪辑` | `本章采用交叉剪辑` |
| `必须采用平行叙事` | `本章采用平行叙事` |
| `时间跳转安排（必须按此执行）` | `时间跳转安排` |
| `【违反此结构要求 = 本章不合格】` | 删除 |
| `【人名硬约束】人物名必须使用以上名称，禁止改名…` | `现有人物：{char_names}` |
| `【叙事风格（必须严格遵守）】` | `【叙事风格】` |
| `【悬念线（必须关注）】` | `【悬念线】` |
| `【本章大纲约束（必须严格执行）】` | `【本章大纲约束】` |
| `严格遵守风格指南。保持…` | `保持…` |
| `本章必须写满2500-3500字` | `本章目标2500-3500字` |

### 3. `prompts.py` — 3项变更

- `## 本章大纲约束（必须严格遵守）` → `## 本章大纲约束`
- 追加隐含解决检测指导（3条判断标准）
- 删除"只提取明确的悬念变化，不要过度解读"

### 4. `cli.py` — 管线集成

**write/loop命令**：generate_chapter()后插入校验块：
- 人名自动修复（`[FIX]` 日志）
- 风格/篇幅警告（`[WARN]` 日志）
- 结构问题重试1次（`[RETRY]` 日志）

**新增 `audit-threads` 命令**：写完全部章节后运行，批量检测未解决悬念线的隐含解决。

### 5. `graph.py` — 2个新方法

- `get_all_threads()`：获取项目所有悬念线
- `update_suspense_thread()`：更新悬念线状态

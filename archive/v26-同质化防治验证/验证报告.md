# V26 验证报告：目的/结尾同质化防治

## 验证目标

V24（100章都市修仙《都市仙尊》）暴露：零大纲模式下98/100章目的相似+98/100章结尾重复。本次验证两层防治方案的有效性。

## 架构变更

### Layer 1：derivation prompt注入"避同"约束
- `prompts.py` ARC_DERIVATION_PROMPT 新增「多样性约束」段落
- 注入近N章的purpose和ending文本，明确要求不重复
- 新增第10条推演规则：purpose/ending必须与近几章不同

### Layer 2：ChapterArc增加ending_type字段
- `ending_type`可选值：cliffhanger(悬念)/revelation(揭示)/quiet(安静)/reversal(反转)/open(开放式)/daily(日常)
- `_check_ending_repetition`升级：优先用ending_type做精确检测（连续3章同类型=high severity），无ending_type时回退到bigram
- MCP工具签名同步更新

### 核心验证指标

#### 1. ending_type多样性 — 通过
**6/6种不同类型，零重复。** 对照V24（98/100章结尾重复），防治效果显著。

#### 2. purpose多样性 — 通过
6章purpose语义全部不同：日常建立→主动调查→认知反转→人物聚焦→物理探索→真相公开+回归。叙事弧线完整。

#### 3. analyze_pacing — 通过
- **零结尾重复检出**（V24对比：96/100章检出）
- 3个低严重度问题：1个目的相似（检测粒度偏紧）、2个节奏剧烈波动（属刻意设计：reversal→quiet→ascending）
- ending_type检测未触发任何high severity告警

#### 4. 悬念线 — 通过
- 7悬念线，4/7解决（57%），3条开放式结尾
- 高优先级悬念线全部推进或解决
- ST01_01（三年前案件）：planted→resolved（Ch6）
- ST03_01（赵维消失）：planted→resolved（Ch6）
- ST01_03（威胁还是保护）：planted→resolved（Ch6）
- ST04_01（胶片）：planted→resolved（Ch5）

#### 5. 一致性 — 通过
仅1个低严重度问题：跨空间对话同时关联两个地点（合理叙事）。

## 结论

1. **两层防治方案有效**：Layer 1（derivation prompt避同约束）+ Layer 2（ending_type分类+重复检测）形成双重防护
2. **零大纲模式可用**：在无outline_entry的情况下，6章ending_type零重复、purpose全部不同
3. **ending_type字段实用**：6种类型覆盖了悬疑叙事的主要结尾模式，LLM能正确选择不同类型
4. **检测准确**：_check_ending_repetition升级后优先使用ending_type，精度高于bigram回退
5. **零回归**：43个MCP工具、196测试点全部通过

## 待改进

- analyze_pacing的目的相似检测在6章规模下粒度偏紧（bigram方法对短文本易误报），50章+规模下表现应更准确
- ending_type目前仅6种，复杂叙事可能需要扩展（如：bittersweet、ambiguous、circular）

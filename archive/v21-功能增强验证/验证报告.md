# V21 功能增强验证报告

## 验证目标

在 V20（语义合规+目的检查）基础上，验证4项功能增强在生产创作管线中的表现：

1. **编辑版本管理**：accept_edit 自动快照 + list_edits/rollback_edit 回滚
2. **Prompt/Extraction 落盘**：3种 prompt + extraction 自动保存到磁盘
3. **地点自动注册**：write_extraction 自动注册未声明的引用地点
4. **批量合规检查**：batch_check_outline_compliance 多章 purpose 合并为单次 LLM 调用

## 生成验证

### 原创作品：《铜锈》

旧物修复师方竹收到一座来历不明的座钟，拆解时发现夹层中藏着四十年前的纸条。追踪纸条指向退休钟表匠老廖，揭开一段关于停摆、记忆与修复的故事。

### 项目设定

| 维度 | 内容 |
|------|------|
| 题材 | 当代现实 / 悬疑 / 人文 |
| 规模 | 6章串行闭环 |
| 人物 | 3核心（方竹/陶然/老廖）+ 2次要（卖钟老头/秀兰） |
| 地点 | 3核心（方竹工坊/旧货市场/老廖钟表铺）+ 5自动注册 |
| 主题 | 修复与遗忘 |
| 意象 | 钟表（停摆→修复→重新走动→再次停摆）、铜锈（覆盖→刮开→光泽→再氧化） |
| 风格指南 | 6条（anti_pattern/rhythm_variation/character_voice/dash_restriction/sensory_diversity/show_dont_tell） |

### 大纲

| 章 | 目的 | 结尾锚点 |
|----|------|---------|
| 1 | 起势：收到座钟，发现夹层纸条 | 方竹读出纸条第一行 |
| 2 | 追踪：陶然认出字迹指向老廖 | 陶然指向老巷子 |
| 3 | 受阻：老廖沉默但手抖 | 所有钟声同时停了 |
| 4 | 深入：发现暗室全是停摆的钟 | 方竹理解了停摆的意义 |
| 5 | 转折：老廖说出座钟真相 | 座钟铜质鸣管发出低音 |
| 6 | 结局：修好钟但调慢一分钟 | "钟在走。铜锈还在。光落下来。" |

### V21 新功能验证

#### 1. Prompt/Extraction 落盘 ✅

每章自动生成4个落盘文件：
- `prompts/derivation_chN.txt`（6个）
- `prompts/writing_chN.txt`（6个）
- `prompts/extraction_chN.txt`（6个）
- `extractions/extraction_chN.json`（6个）

共24个文件，全部自动创建，带 `# Persisted:` 时间戳header。

#### 2. 地点自动注册 ✅

write_extraction 在提取过程中自动注册了多处未预先声明的引用地点（如"老廖钟表铺走廊"、"老廖钟表铺工作间"等），type=auto_registered。避免"地点未注册"冲突，无需每次手动 add_location。

#### 3. 编辑版本管理 ✅

对 Ch3 执行事后编辑验证：
- `review_chapter(action="edit")` → 保存 edited text
- `analyze_edit_impact` → 检出3条受影响悬念线、Ch4-6下游大纲标记
- `accept_edit` → 自动创建快照（ch3_20260521_011542_004153，6事件/38关系）
- `rollback_edit` → 成功回滚（restored_events=6）+ 自动创建回滚前快照
- 最终2个快照文件，数据完整

#### 4. 批量合规检查 ✅

- `batch_check_outline_compliance()` 首次调用：batch_purpose=true，6/6 followed
- 编辑+回滚后再次调用：batch_purpose=true，6/6 followed（回滚恢复原始状态）
- purpose 检查确实合并为单次 LLM 调用（而非逐章调用）

# 闭环提取管线验证报告

日期：2026-05-18

## 核心结论

**闭环提取管线设计验证通过。** 从章节文本提取结构化信息回写图谱的管线已验证可行，粒度标定为5-8事件/章。

## 验证方法

以第12章原文（~4000字）为输入，对比：
- V1提取（无粒度控制）：21个事件
- V2提取（场景级粒度）：7个事件
- Ground Truth（手工录入）：3个事件

## 三版对比

| 指标 | V1(21事件) | V2(7事件) | Ground Truth(3事件) |
|------|-----------|-----------|-------------------|
| 事件数 | 21 | 7 | 3 |
| 间隙事件 | 2(含1误判) | 1(正确) | 1 |
| 地点冲突 | 多处 | 0 | - |
| GT事件覆盖 | - | 3/3 | - |
| 额外捕获 | - | +4场景 | - |
| 细节保留 | 散在事件中 | 集中在updates | 手工精选 |

## V1问题（过度提取）

1. **回忆拆分**：渡口一下午的回忆被拆成5个独立事件（父亲/陈屿/苏晚/周老师/自己），应该合并到1个场景事件
2. **心理活动拆分**："理解间隙本质""承认看见不等于做到""追问想做什么""答案浮现"被拆成4个事件，实际是同一个场景内的认知变化
3. **gap误判**："第一次尝到菜的味道"被标为is_gap=true，实际是日常觉醒
4. **虚拟地点**："记忆中"被当作物理地点使用

## V2改进

1. **场景级合并**：同一地点同一时段=1个事件，回忆内容进detail和character_updates
2. **gap修正**：精确定义间隙=意识滑出身体，尝到菜味归为character_updates(state_change)
3. **地点规范**：回忆人物不设location，新地点必须声明
4. **细节不丢失**：回忆中的具体内容（父亲东风卡车、陈屿路边摊、苏晚QQ灰色、周老师河边钓鱼）全部保留在character_updates中

## Ground Truth vs V2

Ground Truth的3个事件过度压缩：
- "渡口坐一下午后间隙没来" — 实际包含封控解封+面临决定+渡口回忆+做出选择四个阶段
- 遗漏了回家修椅子+尝菜味、离别清晨等重要日常场景

V2的7个事件在GT基础上多捕获4个场景，且通过character_updates保留了所有关键细节。

## 提取管线架构

```
章节文本 → 构建prompt(含已有人物列表) → LLM提取JSON
    → 冲突检测(5类) → 写入图谱 → 生成报告
```

### 冲突检测（5类）

| 类型 | 严重度 | 说明 |
|------|--------|------|
| 事件ID重复 | high | 防止重复写入 |
| 地点未注册 | medium | 未在图谱中且未声明为新地点 |
| 人物未注册 | high | 引用了不存在的人物 |
| 事件过多/过少 | low/medium | >10或<3个事件 |
| 间隙误判 | medium | 检测"尝到/味道/觉醒"等误判关键词 |

### Prompt模板核心规则

1. **合并规则**：同地点同时段=1事件，回忆进detail
2. **拆分规则**：不同地点=不同事件，独立gap=独立事件
3. **细节归档**：回忆内容→character_updates，象征→motif_mentions
4. **间隙定义**：滑出身体+从外部看自己，排除日常觉醒/情感波动/走神
5. **地点规范**：回忆人物不设location，名称一致

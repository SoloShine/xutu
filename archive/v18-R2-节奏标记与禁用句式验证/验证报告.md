# V18-R2 节奏标记与禁用句式验证报告

## 验证目标

测试两个新功能对AI味的进一步改善效果：(1) forbidden_patterns后置校验（"不是X是Y"句式从每章≤1次收紧至0次）；(2) rhythm字段（tight/ascending/descending/mixed）对叙事节奏的显式控制。与V18-R1对比前6章。

## 核心变更

| 参数 | V18-R1 | V18-R2 |
|------|--------|--------|
| forbidden_patterns | 无（仅prompt文本威慑） | 5条正则模式+后置校验 |
| rhythm字段 | 无 | 6条弧线各含rhythm标记 |
| 风格指南数量 | 12（实际13条含detail_economy） | 13（同V18-R1） |
| prev_text_chars | 500 | 500（同V18-R1） |

### 新增功能

1. **forbidden_patterns配置项** — project.yaml中声明禁用句式列表，validators.py后置校验error级别
   - "不是X。是Y。"句式 → max_per_chapter: 0
   - "眼眶热/红了" → max_per_chapter: 0
   - "荧光灯嗡嗡" → max_per_chapter: 0
   - "他在想。" → max_per_chapter: 0
   - "手抖/握紧/指节发白" → max_per_chapter: 1

2. **rhythm字段** — add_chapter_arc新增可选rhythm参数，注入写作prompt
   - tight：紧凑短促，适合追逐/紧迫
   - ascending：递进加速，适合铺垫→高潮
   - descending：递减放慢，适合高潮后沉淀
   - mixed：自由变化，快慢交替

### 代码变更

| 文件 | 改动 |
|------|------|
| config_loader.py | DEFAULTS新增 `validation.forbidden_patterns: []` |
| validators.py | 新增 `validate_forbidden_patterns(text, config)` 函数 |
| main.py | `build_writing_prompt` 新增rhythm指导注入 |
| server.py | `add_chapter_arc` 新增 `rhythm` 参数 |

## 关键发现

### 1. forbidden_patterns后置校验完全消除目标句式

V18-R1的纯prompt威慑（anti_pattern规则中"每章≤1次"）产生13次"不是X是Y"句式。V18-R2改为project.yaml配置max_per_chapter=0 + validators.py后置校验error级别，6章全部0次。**prompt威慑无效，配置+校验有效。**

### 2. rhythm字段对叙事节奏有显式引导效果

6章的实际节奏均与rhythm标记一致：ascending章前慢后快，descending章前快后慢，tight章短句密集场景快速切换，mixed章快慢交替。节奏多样性明显高于V18-R1。

### 3. 破折号使用略有增加

V18-R2共9个破折号（V18-R1仅2个），但仍在每章≤8次限制内。增加集中在Ch3（3个）和Ch4（2个），这两章节奏标记为tight/mixed，破折号用于突然打断的语境合理。

### 4. 串行管线稳定

6章串行写作零中断，每章写作+校验+提取约3-5分钟。字数波动从V18-R1的1.7x降至V18-R2的1.5x（3,938-5,912字）。

### 5. 感官多样性持续生效

6章共使用30+个独立感官细节，章节间零重复。sensory_diversity规则与detail_economy规则协同良好。

## 待改进

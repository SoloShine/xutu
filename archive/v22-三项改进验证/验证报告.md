# V22 三项改进验证报告

## 验证目标

在 V21（功能增强）基础上，验证3项代码改进的稳定性和兼容性：

1. **rollback needs_revision 清理**：rollback_edit 时自动清除下游大纲的 `needs_revision` 标记
2. **Purpose SHA256 缓存**：对 purpose 文本做 SHA256 哈希缓存，4个失效点（revise_outline/write_extraction/clear_chapter_data/rollback_edit）自动使缓存失效
3. **篇幅校验增强**：validate_chapter 的篇幅报告新增 deficit%、场景压缩建议、max_words 信息

## 架构变更

### 修改函数

| 函数 | 文件 | 变更 |
|------|------|------|
| `rollback_edit()` | core.py | 合并后清除下游大纲 needs_revision 标记 |
| `check_outline_compliance()` | core.py | purpose 缓存：SHA256 哈希比对，命中跳过 LLM 调用 |
| `batch_check_outline_compliance()` | core.py | 同上，批量版本也走缓存 |
| `revise_outline()` | kg_json.py | 修改 purpose 时清除缓存 |
| `write_extraction()` | kg_json.py | 写入提取数据时清除缓存 |
| `clear_chapter_data()` | kg_json.py | 清除章节数据时清除缓存 |
| `restore_snapshot()` | kg_json.py | 恢复快照时清除缓存 |
| `validate_chapter()` | core.py | 篇幅报告增强：deficit%、场景压缩、max_words |

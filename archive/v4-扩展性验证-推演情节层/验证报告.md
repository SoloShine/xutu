# 扩展性验证报告：推演情节层

日期：2026-05-18

## 核心结论

**从图谱数据推演新章节的ChapterArc可行。** 234字的推演arc使续写的结尾锚点从"对但模糊"变为"精确命中"。

## 验证方法

A/B对比：写假设的第13章（原小说12章完结）
- A组：结构层+风格层（无ChapterArc，1497字prompt）
- B组：结构层+风格层+推演ChapterArc（1731字prompt，多234字）

## ChapterArc推演方法

从图谱查询"推演上下文"：
1. 近3章的ChapterArc（叙事轨迹）
2. 近3章的Event（含type/is_gap/detail）
3. 人物状态（通过事件关联）
4. 主题线（惯性→渺小→选择）
5. 意象演变（evolution字段）
6. 最后一个Event（PRECEDES链末端）

推演出的ch13 arc：
```json
{
  "purpose": "回到城市面对旧程序，第一次主动重启——选择的检验",
  "scenes": "大巴到达城市 → 出租屋 → 公司办离职 → 路过修理店停步",
  "ending": "站在店门口看里面的人做具体的事，没进去，站了一会儿",
  "gap_note": "本章没有间隙。注意力从内部转向外部"
}
```

## 对比结果

| 维度 | A组(无arc) | B组(推演arc) |
|------|-----------|------------|
| 核心美学(不宣告) | 通过 | 通过 |
| 情节方向 | 对但自由散漫 | 精确跟随arc |
| 对话风格 | 通过 | 通过 |
| 间隙处理 | 正确(无间隙) | 正确(无间隙) |
| 意象运用 | 有但弱 | "像父亲修车铺"精准呼应 |
| 结尾锚点 | 日常感但模糊 | "不快不慢"精确命中速度意象 |

篇幅：A组~950字，B组~750字（arc控制更紧凑）

## 关键发现

1. **风格层保底有效**：两组都遵守了风格规则，说明结构层+风格层足以保证美学质量
2. **情节层的作用是精度而非方向**：没有arc也能大致走对方向，但有arc才能精确命中结尾
3. **推演上下文来自图谱**：不需要外部输入，完全从已有图谱数据推演
4. **新增方法**：`graph.py:get_arc_derivation_context()` + `prompts.py:ARC_DERIVATION_PROMPT`

## 新增文件

| 文件 | 用途 |
|------|------|
| graph.py `get_arc_derivation_context()` | 查询推演上下文 |
| prompts.py `ARC_DERIVATION_PROMPT` | 推演prompt模板 |
| derived_arc_ch13.json | 推演出的ch13 ChapterArc |
| test_extensibility.py | A/B对比测试脚本 |

## 局限

- ch13是假设续写，无ground truth对照
- 篇幅偏短(~750-950字 vs 原文~4000字/章)
- 推演用的是人工模拟LLM，未测试实际LLM推演质量

# V20 语义合规+目的检查验证报告

## 验证目标

1. **LLM语义合规检查**：补充bigram匹配无法识别的语义等价表述（如"深夜来访"="突然出现"）
2. **目的级别合规检查**：新增purpose_alignment检查维度，检测叙事目的层面的偏离
3. **V17 50章回溯验证**：用V17已有数据验证合规检查在长篇下的偏离检测能力

## 架构变更

### 三步合规检查管线

```
Step 1: 程序化bigram匹配（快速、零成本）
  ↓ 未匹配项
Step 2: LLM语义等价判定（按需触发）
  ↓ 所有章节
Step 3: LLM目的合规检查（始终执行）
```

### 新增函数

| 函数 | 用途 |
|------|------|
| `_semantic_compliance_check()` | LLM语义等价判定（bigram fallback） |
| `_purpose_compliance_check()` | LLM目的合规检查（独立维度） |

### validators.py变更

- `check_outline_compliance`的severity从`error`/`warning`改为`pending_semantic`
- 程序化检查不再直接判定失败，而是标记为`pending_semantic`等待LLM第二步判定
- `structure_hint`检查保持`warning`（不受语义检查影响）

### config_loader.py变更

```python
"collaboration": {
    "semantic_check": True,  # 新增：控制LLM语义检查
}
```

## 端到端验证

### 验证1：V19 Ch5语义检查

**问题**：大纲key_events="宋姐突然出现"，实际事件="宋姐深夜来访"，bigram不匹配。

| 检查步骤 | 结果 |
|---------|------|
| Step 1 bigram | 未匹配（pending_semantic） |
| Step 2 LLM语义 | ✅ 匹配 — "深夜来访"与"突然出现"语义等价 |
| Step 3 目的检查 | ✅ aligned |
| **最终** | **followed** |

### 验证2：V17 50章回溯全量扫描

| 维度 | 结果 |
|------|------|
| 总耗时 | 769秒（15.4秒/章） |
| aligned | 31章 |
| diverged | 19章（全部confidence=high） |
| unknown | 0章 |

**偏离章节分布**：

```
Ch1-26   ✅ 全部aligned（26章连续对齐）
Ch27     ❌ diverged — 首次偏离点
Ch28-29  ✅ aligned（短暂回归）
Ch30-44  ❌ diverged（15章连续偏离）
Ch45     ✅ aligned（"零号学说话"弧线回归）
Ch46-48  ❌ diverged（3章）
Ch49-50  ✅ aligned（终章回归）
```

**与V17原始报告的对应**：

V17报告称"大纲从Ch27起完全偏离"。目的级别检查精确命中：
- 首次偏离点Ch27 ✅
- 持续偏离区间Ch30-48（中间有Ch45回归）✅
- 终章回归Ch49-50 ✅

**Ch28-29的"回归"**：这两章的大纲目的是"盟友集结"和"潜入准备"，实际事件虽然细节不同但叙事方向一致（团队组建相关），LLM正确判定为aligned。

### 验证3：bigram vs purpose检查对比

| 章节 | bigram | purpose | 实际情况 |
|------|--------|---------|---------|
| Ch1-26 | 全部followed | 全部aligned | ✅ 未偏离 |
| Ch27 | followed | **diverged** | ✅ 偏离 |
| Ch30 | followed | **diverged** | ✅ 偏离 |
| Ch31 | followed | **diverged** | ✅ 偏离 |
| Ch34 | followed | **diverged** | ✅ 偏离 |

**结论**：bigram匹配在V17的长描述式key_events下完全失效（50/50 passed），而purpose检查精准检测出19/50的偏离章节。

### 关键洞察：大纲条目粒度

| 维度 | V17 key_events | V19 key_events |
|------|---------------|---------------|
| 格式 | 完整场景描写（含对白、动作） | 简洁事件标签 |
| 长度 | 100-200字/事件 | 5-10字/事件 |
| bigram有效性 | 无效（假followed） | 有效 |
| purpose检查 | 有效 | 有效 |

**建议**：大纲key_events应使用简洁标签（如"发现设备"、"宋姐出现"），而非场景描写。purpose检查是最终的安全网，无论key_events格式如何都能检测偏离。

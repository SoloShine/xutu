# V24 100章图谱自演化验证报告

## 验证目标

验证一个更激进的假设：**完全不用大纲，让故事由图谱状态自驱动演化，100章自动收束到结局。**

这是对"故事是数据结构"这一核心理念的终极验证——如果图谱足够丰富，不需要人为规划情节，系统能自己长出一个完整故事。

**结论：验证通过。100章闭环零中断，5条高优先级悬念线全部解决，故事自然收束。**

## 技术方案

### 零代码变更

现有 MCP 工具完全支持无大纲生成：
- `get_derivation_prompt`：outline 为空时，纯靠近3章弧线+事件+悬念线驱动推演
- `get_writing_prompt`：无 outline_entry 时不注入大纲约束
- `write_extraction`：自动注册新人物/地点

### 每章流程（串行，子代理执行）

```
for ch in 1..100:
    1. get_derivation_prompt(project, ch)  → 推演上下文
       LLM 生成 arc JSON
    2. add_chapter_arc(project, ch, ...)   → 写入弧线
    3. get_writing_prompt(project, ch)      → 写作上下文（含前章结尾500字）
       LLM 生成正文 (~5000-9000字)
    4. 保存到 output/chN_generated.txt
    5. get_extraction_prompt(project, ch, text) → 提取上下文
       LLM 生成提取 JSON
    6. write_extraction(project, ch, json)  → 写回图谱
```

### 收束机制（纯数据驱动）

1. **风格指南第10条**（convergence）：注入每章写作prompt，Ch60+优先收线，Ch80+只解决不新开
2. **悬念线预算**：max(8, ch*1.5)，控制并行活跃线索数
3. **弧线推演 lookback=3**：自动感知最近几章趋势
4. **前章注入 prev_text_chars=500**：感官/风格连续性

## 已知限制

1. **目的同质化**：98/100章触发"目的相似"检测。LLM在无大纲约束下，弧线推演的purpose高度模式化（"揭示真相""对抗敌人""修炼突破"循环）。这是LLM生成pattern固化的已知问题
2. **结尾同质化**：98/100章触发"结尾重复"检测。都市修仙场景导致结尾高频出现"蹲/站/半跪"等姿势描写+灵力状态描写。需要更丰富的结尾锚点控制
3. **地点爆炸**：104个地点（vs V17的14个）。大量一次性场景（废弃工厂、旧楼、管道等）被自动注册为地点。不影响功能但增加图谱体积
4. **前半程节奏波动**：Ch1-20有多处节奏剧烈波动（相邻章强度差>5），打脸密集导致节奏不够均匀
5. **API内容安全**：中国LLM API（智谱）对战斗场景触发内容过滤，需指导子代理使用抽象战斗描写

## 核心结论

**"故事是数据结构"这一假设在100章规模下得到验证。**

- 不使用任何大纲，仅靠7个人物+5个地点+5条悬念线+10条风格指南的初始图谱，系统自动演化出100章完整故事
- 5条高优先级悬念线通过图谱状态追踪和收束风格指南自然解决，无需人工干预
- 故事经历了觉醒→升级→真相→收束的完整弧线，与经典都市爽文结构吻合
- 图谱增长健康：515事件/118悬念线/2730关系，query性能稳定

**V17 vs V24的核心差异**：V17使用了50章大纲（Ch27后大纲偏离但系统自适应），V24完全不使用大纲。两者都实现了100%闭环，但V24的故事结构更"自然涌现"，中期有更多探索性章节（灵脉暗河Ch20-40），节奏不如V17紧凑但更有机。

# V9 原创创作验证报告

## 验证目标

验证系统辅助**完全原创创作**的能力：从零开始，无源文本可提取，仅凭用户200字的创作方向描述，LLM自动生成世界观、逐章续写、提取回写，完成一部6章原创短篇。

这是系统从"分析续写工具"到"原创创作工具"的关键门槛。

## 验证方法

1. 用户输入创作方向（约200字）："超现实主义悬疑小说，关于记忆与身份"
2. `setup --auto`：LLM生成完整世界观（角色/地点/主题/风格指南/时间段/第1章arc）
3. `write 1 --auto`：生成第1章（空图谱，无前文事件）
4. `extract 1 --auto`：提取第1章结构化数据回写图谱
5. `loop 2-5 --auto`：自动循环（提取→推演→续写），完成ch2-ch6
6. 评估6章整体质量

## 作品概况

**标题**：错轨夜车
**类型**：超现实主义悬疑
**核心概念**：集体失眠城市中，出租车司机林生发现自己正承载他人错乱的记忆碎片，而这些碎片指向一个被遗忘的溺水事件——他自己的。

### 世界观设定

| 维度 | 内容 |
|------|------|
| 角色 | 林生（主角/出租车司机）、陈姐（记忆载体）、吴明（极端理智者）、小夏（幻影） |
| 地点 | 永昼环岛、废弃旱冰场"漩涡"、尾号444出租车内部、南山疗养院旧址 |
| 主题 | 记忆的所有权（1-6章）、逃避的代价（1-4章） |
| 风格指南 | 7条（详见world_setup.json） |
| 时间段 | 潜伏期(ch1-2) → 融合期(ch3-4) → 崩溃期(ch5-6) |

### 风格一致性

7条风格指南全部被严格遵守，无一违反：

| 规则 | 遵守情况 |
|------|---------|
| 禁止内心独白解释机制 | 通过环境细节（探照灯、无影人、黑眼圈）展现世界观 |
| 乘客逻辑错位对话 | 陈姐用天气预报语气讲述溺水、吴明播报死亡参数 |
| 通感修辞 | "听到霓虹灯的腥味""腥臭味在耳膜上刮擦出金属音" |
| 无明确梦境边界 | 场景变换通过物理变形完成（隧道→大坝→水下） |
| 车厢空间收缩 | 从宽敞到贴脸→肉囊→完全淹没，逐章升级 |
| 对话模糊化 | 无直接提问"这是你的记忆吗"，通过动作（摸不存在的伤疤）确认 |
| 结尾悬而未决 | 每章结尾停留在反常画面（浮鞋/无脸脸/挥手/笔尖/30%/微笑） |

### 人物一致性

- **林生**：从麻木克制的旁观者，逐步被记忆侵蚀——双手透明化、身体下沉、最终被"容器化"。人物弧线完整
- **吴明**：极端理智者，从观察记录者（后座）到主动执行者（强行握笔签名），角色功能随情节推进升级
- **陈姐**：在ch1-2完成记忆载体的功能后淡出，不强行维持出场
- **小夏**：setup中定义但未在6章中实质出场——弧线窗口不足以展开所有角色

### 叙事递进

```
潜伏期(ch1-2)：日常怪异 → 记忆入侵（浮鞋/隧道/水渍）
融合期(ch3-4)：物理同化 → 身体异变（鞋带上脚/死亡通报/透明手）
崩溃期(ch5-6)：身份覆写 → 强制签名 → 7岁溺亡男孩的微笑
```

三阶段递进清晰，与setup定义的时间段主题完全吻合。

### 漂移评估

**短期漂移**：无。3-arc滑动窗口保持了章间连贯性：
- 每章的结尾状态被下一章的arc精准衔接
- 意象（水草/皮鞋/透明化）逐章递进而非凭空出现
- 吴明从观察者→执行者的角色转变有明确的中间过渡

**长期漂移风险**：可控但存在。
- 核心主题"记忆的所有权"贯穿6章，未偏移
- 但setup中定义的**小夏**角色和**南山疗养院**地点未被使用——说明滑动窗口缺少全局规划视角
- 6章范围内可接受，超过10章可能出现更明显的偏移

## 结论

**完全原创创作验证通过。**

核心发现：

1. **世界观生成是可行的起点**：LLM从200字方向描述生成的世界观（角色/地点/风格/主题/时间段）足以驱动6章连贯创作
2. **风格指南是最强的约束层**：7条具体风格规则被6章严格遵循，是维持一致性的最关键因素——与v8边城验证结论一致
3. **滑动窗口在6章范围内足够**：3-arc窗口维持了章间连贯，无短期漂移
4. **长期规划缺失是已知限制**：setup中定义的部分内容（小夏/南山疗养院）未被展开，需要全局大纲补位
5. **篇幅控制仍需改进**：ch1和ch3偏短（~2200-2800），ch2偏长（5500），与目标2500-3500有偏差

## 冷启动原创创作流程（验证通过）

```
1. 准备创作方向描述（~200字）
2. python cli.py -p <project> init
3. python cli.py -p <project> setup --direction "描述" --auto
4. python cli.py -p <project> write 1 --auto
5. python cli.py -p <project> extract 1 <ch1_text> --auto
6. python cli.py -p <project> loop 2 <ch1_text> --auto  # 或分步 derive + write
   ... 重复到目标章节数 ...
```

总人工输入：~200字（创作方向）。其余全自动。

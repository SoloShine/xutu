## 遥测设计

### 零侵入装饰器

`telemetry.wrap(func)` 装饰器包裹core.py的43个公共函数：

```python
# core.py 底部，模块加载时自动执行
import telemetry as _telemetry
for _fname in _PUBLIC_TOOLS:
    if _fname in globals():
        globals()[_fname] = _telemetry.wrap(globals()[_fname])
```

### 懒初始化

首次 `_kg(project)` 调用时初始化 `TelemetryCollector`，进程级单例：

```python
def _kg(project: str):
    if project not in _pool:
        _pool[project] = _create_backend(project)
        import telemetry as _tel
        if _tel._collector is None:
            _tel.init_telemetry(project)
    return _pool[project]
```

### 自动保存

`write_extraction` 完成后自动调用 `save_chapter_report(chapter)` 将遥测数据写入磁盘：

```python
# wrap() 的 finally 块
if func.__name__ == "write_extraction" and chapter is not None:
    _collector.save_chapter_report(chapter)
```

### LLM Token捕获

`llm._last_usage` dict 在非stream API调用后捕获 `response.usage`，遥测装饰器读取后清空：

```python
# llm.py
if hasattr(response, 'usage') and response.usage:
    _last_usage.update({
        "prompt_tokens": response.usage.prompt_tokens or 0,
        "completion_tokens": response.usage.completion_tokens or 0,
    })
```

## 已知限制

1. **自动保存bug**：`write_extraction`完成后的自动保存未触发（原因待查）。临时方案：主会话手动调用`save_telemetry_chapter_report`。
2. **决策提取全空**：`add_chapter_arc`和`write_extraction`的decision字段为空。可能原因：MCP/FastMCP参数传递层做了一层转换，kwargs不含原始参数。
3. **LLM token**：子代理的LLM调用不经过MCP server的llm.py，token消耗无法被telemetry捕获。
4. **Ch1数据丢失**：重启MCP server后collector数据清空，ch1遥测数据丢失。需改为实时追加写入。
5. **禁用句式不完美**：R2仍有1处情感类违规（"不是愤怒，不是威胁"）。prompt威慑效果显著（46→1）但不100%可靠。
